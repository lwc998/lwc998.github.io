浏览器的事件循环

任务队列

  • js 引擎遇到异步任务时，不会一直等待其返回结果，而是将该任务放入任务队列中，等待执行

事件循环

  • 浏览器的引擎会一直轮询任务队列，当任务队列中有任务时，会从队列中取出任务，执行任务，当任务执行完毕，会返回任务队列，继续执行任务队列中的任务，直到任务队列为空，才进入下一轮循环

执行顺序

  • 执行宏任务，然后执行该宏任务产生的微任务，若微任务在执行过程中产生了新的微任务，则继续执行微任务，微任务执行完毕后，再回到宏任务中进行下一轮循环

宏任务和微任务

  • 宏任务：script，setTimeout，setInterval，setImmediate，I/O，UI rendering
  • 微任务：process.nextTick，new Promise().then(callback)、MutationObserve 等（async 和 await）实际上是 Promise 的语法糖

练习题

console.log("start");
setTimeout(() => {
  console.log("children2");
  Promise.resolve().then(() => {
    console.log("children3");
  });
}, 0);

new Promise(function (resolve, reject) {
  console.log("children4");
  setTimeout(function () {
    console.log("children5");
    resolve("children6");
  }, 0);
}).then((res) => {
  console.log("children7");
  setTimeout(() => {
    console.log(res);
  }, 0);
});

// promise状态改变后 才执行then回调
// 输出

// children4
// children2
// children3
// children5
// children7
// children6

  return new Promise((resolve, reject) => {

      resolve(2);

    p1.then((res) => {
      console.log(res);

p().then((res) => {
  console.log(res);

async function errorFunc() {
  try {
    // Tips:参考：https://zh.javascript.info/promise-error-handling：隐式 try…catch
    // Promise.reject()方法返回一个带有拒绝原因的Promise对象
    // Promise.reject('error!!!') === new Error('error!!!')
    await Promise.reject("error!!!");
  } catch (e) {
    console.log(e);
  }
  console.log("async1");
  return Promise.resolve("async1 success");
}
errorFunc().then((res) => console.log(res));

function bar() {
  console.log("async2 end");
}

console.log("script end");

// promise1

// error!!!

// promise2
