前端性能优化

1、前置知识

面试题：从输入 URL 到页面渲染的过程

  1. DNS 解析
  • 从域名中提取主机名，并通过 DNS 域名系统解析对应的 IP 地址
  • 优化方式
    • 使用 DNS 预解析技术，在 HTML 头部使用<link rel="dns-prefetch">来预解析域名
    • 将 DNS 解析结果缓存到本地，减少 DNS 查询次数
  1. TCP 连接
  • "三次握手"，通过发送和接受 SYN 和 ACK 数据包来确保连接的建立
  • 优化方式
    • 使用 HTTP2 协议，该协议支持多路复用，可在同一个 TCP 连接上并行处理多个请求，提高传输效率
    • 使用持久连接(Keep-Alive)，避免频繁建立和关闭 TCP 连接
  1. 发送 HTTP 请求
  • 优化方式（网络传输性能优化）
    • 合并小文件，减少请求次数
    • 启用浏览器缓存
    • 压缩请求和响应数据的大小
      • 使用 gzip 压缩，减少传输数据大小
      • 通过构建工具自动对代码进行转换、压缩和合并
    • 使用 CDN 加速静态资源的加载

  1. 服务器处理请求并返回处理结果

  2. 浏览器进行解析

  • 解析 HTML 文档，生成 DOM 树
  • 解析 CSS，生成 CSS 规则树
  • 关联 DOM 树和 CSS 规则树，生成渲染树(render tree)
  • 布局渲染树，对各元素的尺寸和位置进行计算
  • 绘制渲染树，绘制页面的像素信息
  • 优化方式（页面渲染性能优化）
    • 异步加载 JavaScript 和 CSS，避免阻塞页面渲染
    • WebP、jpg 图片压缩
    • 使用懒加载技术，对图片等资源进行延迟加载，提高页面初始加载速度
    • 优化 CSS 选择器的性能，避免使用复杂的选择器
    • 减少重排重绘
      • 重排：改变 DOM 的位置或大小，导致浏览器重新生成渲染树
      • 重绘：所有对元素的视觉表现属性的修改
      • 减少页面 DOM 操作
      • 合理使用防抖和节流
  1. 连接结束关闭 TCP 链接（四次挥手）
