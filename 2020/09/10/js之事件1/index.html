各种事件及一些算法思想

  1. 表单：value改变表单里的值：input.value = “被点击了”; 禁用表单：disabled = true;

    this.disabled = true; this是指向事件函数的调用者

  2. 样式：1）行内样式：element.style = ‘’;2)类名样式：element.className

  3. 注册事件：1） 获得焦点事件：onfocus; 失去焦点事件：onblur;

    ​ 2）移动或离开鼠标：onmouseover 和 onmouseout;

    ​ 3）按住或松开鼠标：onmousedown和onmouseup;

    ​ 4）点击：onclick

  4. 排他思想：

  1. 添加类名：btn.onclick = function{

    //直接覆盖原先的 this.className = ‘chage’;

    //不覆盖原先的 this.className = ‘原先 change’;}

  2. 复选框选中的状态：this.check,返回true为选中；false未选中

  3. 属性的操作：1）获取属性：element.属性——获取内置属性

    ​ element.getAttribute(‘属性’)——获取自定义属性

（H5自定义属性名用data-开头；用驼峰命名）——获取用div.dataset.属性名或div.dataset.[‘属性’]

​ 2）设置属性：element.属性 = ‘值’、class类名时用classname

​ element.setAttribute(‘属性’，’值’) 主要用于自定义属性

​ 3）移除属性：element.removeAttribute(‘属性’)

  1. 除了innerHTML,也可用childNodes和nodeValue属性来获取元素

  1. 三种创建元素方式区别：1）document.write():创建元素（如果页面流动加载完毕，再调用导致重

    ​ 绘，之前加载的都没了

    ​ 2）element.innerHTML():创建多个元素效率高（不拼接字符，采取数组 形式拼接）

                                         3）createElement()：创建多个元素效率低一点，但结构比较清晰

节点操作

  1. 父节点parentNode :erweima.parentNode(没用最近的父亲返回null)

  2. 子节点

    1. childNodes 返回包含所有节点（包括元素、文本节点）
    2. children 返回元素节点
    3. parentNode.firstChild/lastChild 返回元素、文本节点
    4. parentNode.lastElementChild / firstElementChild 返回元素节点 ie9才支持
    5. 实际开发parentchildren[0]

  3. 兄弟节点：

    1. 下一个兄弟节点：nextSibling 包括所有节点 元素(换行/空格)、文本
    2. nextElementSibling:只有元素节点、previousSibling: ie9以上兼容

  4. 创建节点：document.createElement(‘tagName’); 创建之后要进行添加操作才能显示

  5. 添加节点 从后面追加：node.appendChild(child) :ul.appendChild(li) （node父亲、child儿子）

  6. 添加节点 从前追加：node.insertBefore(child追加的,指定元素ul.children[0])

  7. 删除节点：node父亲.removeChild(child)

  8. 复制节点：node.clone():括号内为空或false为浅拷贝，只复制标签、不复制内容；括号内为true为深拷贝
